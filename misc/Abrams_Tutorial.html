10 Sessions

Each block runs own interpreter session every time it is evaluated. Use session: header parameter to run all blocks in the same interpreter session 

avar = 42
return avar

return avar / 2

#RESULTS: Traceback (most recent call last): File "<stdin>", line 5, in <module> File "<stdin>", line 3, in main NameError: global name 'avar' is not defined #ENDRESULTS 

ave = 42

ave / 2

You can also switch to the foobar buffer to interact with interpreter, et c.

Warning: A :session setting for a section is shared for each block… regardless of language! Probably not what you want.

13 Literate Programming

13.1 Tangling

  • :tangle parameter takes specified blocks & puts them in a source file 
    while s = gets
  print s ; puts
end
  • C-c C-v t renders file
  • Blocks are included in file in order.
  • :tangle yes writes blocks to file with same name as org file
  • For entire file: #+PROPERTY: tangle ./bling-mode.el

13.2 Comments

  • Have prose turned into comments: Prose above a block is rendered as comments in the tangled source file.
  • done by setting the #+PROPERTY: comments org

13.3 Shebang

The :shebang parameter specifies interpreter to use. 

while s = gets
  puts "#{$<.file.lineno}: #{s}"
end

13.4 Noweb

puts $F.last

ruby -ane 'puts $F.last'

13.5 Variables

Pass values into source block!

[a, b, c, d, e, f, g ]
